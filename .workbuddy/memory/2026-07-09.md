## 修复 GET /api/passwords 500 + 登录 409（第 8 项改动）
- **500 根因**：部署库由旧版镜像创建，`passwords` 表缺 `deleted` 列（软删除），`list_passwords` 的 `filter_by(deleted=False)` → `sqlalchemy.exc.OperationalError: no such column: passwords.deleted` → 500。原 `db._migrate_columns()` 是**硬编码列清单，漏了 `passwords.deleted`**（以及 `history.changed_by/comment`）。
- **修复**：把 `_migrate_columns` 改为**通用迁移**——扫描 `Base.metadata` 所有模型列，对每张已存在的表用 `ALTER TABLE ADD COLUMN` 补齐缺失列（按 SQLAlchemy 类型映射 SQLite DDL，并带安全默认值：模型标量默认优先，否则 NOT NULL 给类型兜底默认）。`init_db()` 在 `seed.seed()` 启动时调用，自动修复旧库。**此法一劳永逸**，以后加列无需再手维护清单。
- **409 根因（真正阻塞登录）**：`login/begin` 对未迁移账号返 409，前端 `doLogin` 本应回退明文 `/login`；但 `api()` 抛的 `Error.message` 是中文 `detail` 文本（"账户尚未启用加密登录…"），**不含 "409"**，所以 `err.message.includes("409")` 判断失败 → 回退不触发 → 用户卡在 409 错误、无法登录。
- **修复**：`api()` 抛错时 `e.status = res.status` 附上状态码；`doLogin` 改判 `err.status === 409 || String(err.message).includes("409")`，回退路径恢复。后端 409 行为不变（仍是设计内的迁移触发）。
- **验证（端到端）**：起 `pm-fix`(9016) 建数据→停容器 `ALTER TABLE passwords DROP COLUMN deleted` 模拟旧库→重启**未修复**镜像复现 500（traceback 正是 `no such column: passwords.deleted`）；重建镜像后同卷重启，启动日志无报错、`deleted` 被补回且旧行 `deleted=0`（数据保留），`GET /api/passwords`→200 且返回原条目。另直接 blank 某用户 `pw_verifier` 验证 409→`/login` 迁移→200 全链路。
- 新增 `smoke_migration.py`(纯 stdlib，连 live 容器) 10 项全过：admin 登录、GET /api/passwords==200(500 回归护栏)、建分组+密码可列出、删除后审计 ?action=delete 含账号说明、普通用户访问审计 403。

## 修复 SM2 OrgKey 解密报错：decrypt() got unexpected keyword 'passphrase'（第 11 项改动）
- **根因**：`manager.decrypt_with_orgkey`(manager.py:60) 与导入校验 `_validate_keys`(keys.py:94) 对**所有** provider 统一调用 `.decrypt(ct, priv, passphrase=pp)`。GPG 的 `decrypt` 有 `passphrase=None` 形参，但 **SM2 的 `sm2_crypto.decrypt/decrypt_bytes` 没有该形参** → 用 SM2 的 OrgKey 私钥解密密码条目时抛 `TypeError: decrypt() got an unexpected keyword argument 'passphrase'`，被 passwords.py:186 包装成「用 OrgKey 私钥解密失败：…」500。
- **修复**：给 `sm2_crypto.decrypt` / `decrypt_bytes` 各加 `passphrase: str = None` 形参（**直接忽略**——本系统 SM2 OrgKey 私钥以原始 hex 存储，不支持 passphrase 保护；保留形参仅为兼容 GPG/SM2 统一调用接口）。这是最小且一致的修复，不必在 manager 里按算法分支。
- **验证（容器内端到端）**：起 `pm-sm2fix`(9013) 跑 `smoke_keyimport.py`（在 [6] 新增 SM2 回归：生成 SM2 OrgKey → 用其创建条目 → unlock 读回明文；此前必 500）：**全部通过 ✅**，SM2 OrgKey 解密读回「SM2-中文-secret」匹配。回归 `smoke_migration`(10/10)、`smoke_export`(10/10，注意其默认 BASE=9021，需传 `http://localhost:9010` 参数) 仍全过。
