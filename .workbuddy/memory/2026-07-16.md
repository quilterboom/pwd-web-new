## 操作系统即重算登录失效时间（服务端空闲计时对齐真实操作）
- 需求：用户操作系统时，登录失效时间要重新计算（不能因「纯前端操作不打 API」而被服务端空闲吊销）。
- 根因：前端空闲监听已随活动重置本地 60s 计时；但服务端 `auth_sessions.last_activity` 仅在「带令牌的 API 请求」时刷新（sessions.is_session_valid 节流 15s）。纯前端操作（看/点/滚）不打 API → 服务端 60s 无活动即吊销令牌。
- 后端改动：
  - `backend/app/sessions.py`：新增 `touch_session(db, jti)` → 显式刷新 `last_activity`（已吊销/不存在/无 jti 返回 False）。
