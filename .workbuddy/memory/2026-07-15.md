## 为离线密码管理器启用 HTTPS（自签名 + HTTP 跳转）
- 需求：用户要求访问网址用 HTTPS。内网离线环境，选择：自签名证书（IP+域名都进 SAN）+ 保持 9010 作 HTTPS 端口 + 明文 HTTP 自动 301/307 跳 HTTPS。
- 关键前提（已确认）：
  - `run.py` 原生支持 TLS（设 `SSL_CERTFILE`/`SSL_KEYFILE` 即走 uvicorn TLS），**无需改前端**（前端 `api()` 用同源相对路径 `fetch('/api/...')`，无混合内容问题）。
  - 前端产物 `index-CHiLxpFF.js` 本轮未变（只改了 run.py/main.py/compose）。
- 改动：
  - `backend/run.py`：SSL 启用时除 uvicorn TLS 外，另起后台线程 `ThreadingHTTPServer` 明文服务（端口 `HTTP_PORT` 默认 9080），对所有方法 307 跳 `https://<host>:<PORT>`；`<host>` 取自请求 Host 头（去端口），故 IP/域名都适用。`SSL_REDIRECT`（默认1）可关。
  - `backend/app/main.py`：`security_headers` 中间件新增**可选 HSTS**（仅 `SSL_CERTFILE` 设置且 `HSTS=1` 时下发 `max-age=31536000; includeSubDomains`），默认关，避免自签名轮换被锁死。

  - `backend/.env.example`：补充 SSL 变量说明。`Dockerfile` EXPOSE 改为 `9010 9080`。
- 重建：`bash backend/offline/build_image.sh`（仅重拷 run.py 层，pip 缓存命中，快）。新镜像 `password-manager:latest` = `d726caf223b8`（219MB，10:08）；`backend/offline/password_manager_image.tar` 更新（216M）。

## 登录报 401 排查（用户反馈 admin 登录 401）
- 结论：**登录代码本身无 bug**。用前端原生 `sm3.js` 在干净数据卷 + 正确密码做端到端 SCRAM 登录，`login/begin`→`login/verify` 全程 200 并拿到 JWT；前后端 SM3 字节级对齐，HTTPS 改造不影响登录。
- 401 根因只可能是「密码错」或「旧数据卷账号凭据不兼容」（如旧账号只有 bcrypt `hashed_password` 无 `pw_verifier`，触发 409 回退 `/login` 明文路径）。
- 最经典坑：**升级镜像挂了旧数据卷 + 改了 `ADMIN_PASSWORD`，但 seed 只在用户不存在时跑一次**，已存在 admin 的密码不变 → 用旧密码登录或强重置。
- 强重置 admin 密码命令（容器内，同时写 SCRAM `pw_verifier`+bcrypt `hashed_password`，两种登录路径都通）：
  ```bash
  docker exec -e NEW_PW='新密码' password-manager python - <<'PY'
  import os, secrets
  from app.db import SessionLocal
  from app.security import hash_password, derive_password_verifier
  from app.models import User
  db = SessionLocal()
  u = db.query(User).filter_by(username='admin').first()
  if not u:
      print('admin 不存在（全新部署 seed 未生效，需另查）'); raise SystemExit
  new = os.environ['NEW_PW']
  u.pw_salt = secrets.token_bytes(16).hex()
  u.pw_verifier = derive_password_verifier(new, u.pw_salt)
  u.hashed_password = hash_password(new)
  db.commit()
  print('OK: admin 密码已重置，请重新登录')
  PY
  ```
- 只读诊断（看 admin 是否存在 + 凭据格式）：`docker exec password-manager python -c "from app.db import SessionLocal; from app.models import User; [print(u.username,'admin=',u.is_admin,'SCRAM=',bool(u.pw_verifier),'bcrypt=',bool(u.hashed_password)) for u in SessionLocal().query(User).all()]"`

## 新增「自助注册」功能（开放模型 + 总开关，默认关）
- 需求：用户要求加「注册用户」功能。安全决策：开放自助注册 + 总开关 `ALLOW_REGISTRATION`（默认关）+ 自动加入默认分组 `REGISTER_DEFAULT_GROUP`（默认「默认分组」），注册只建**普通用户**（is_admin=False），由管理员后台分配权限/分组。
- 改动：
  - `backend/app/routers/auth.py`：新增公开端点 `POST /api/auth/register`（入参 `username`/`password`/`confirm_password`）；受 `ALLOW_REGISTRATION` 总开关控制（默认关→403「当前系统未开放注册」）；**独立注册限速** `_register_rate_limit`（默认 1h 内 10 次/IP，与登录限速分开）；校验 用户名≥3位、密码≥8位、两次一致、用户名唯一；创建用户复用 `_seed_login_material` 写 SCRAM(`pw_salt`/`pw_verifier`)+bcrypt(`hashed_password`) 双凭据；自动 find-or-create `REGISTER_DEFAULT_GROUP` 并 `_link_user_group` 加入。
  - `backend/app/config.py`：新增 `ALLOW_REGISTRATION`（默认 `"0"`→False）、`REGISTER_DEFAULT_GROUP`（默认 `DEFAULT_GROUP_NAME`）。
  - 前端 `backend/frontend/src/api/auth.js`：新增 `register(username,password)`；`Login.vue`：登录/注册模式切换 + 注册表单（用户名/密码/确认密码）+ 成功后切回登录并预填。
