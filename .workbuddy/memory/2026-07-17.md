## 生成离线依赖 wheel 包（offline_wheels）
- 用户要在离线电脑上用 VSCode 裸机跑本项目，需提前在有网机下载离线 Python 依赖。
- 产物：`offline_wheels/`（项目根），46 文件 / 20MB，覆盖 **win_amd64 的 Python 3.11 与 3.13**。
- 关键坑：`PGPy==0.6.0` 在 PyPI **仅发布 sdist 源码包**（无预编译 wheel），故走 `PGPy-0.6.0.tar.gz`；离线安装需构建后端 `setuptools/wheel`（已纳入）。
- 下载分工：wheel 层用 `--only-binary=:all: --platform win_amd64 --python-version 31x`（临时清单排除 PGPy）；PGPy 单独 `pip download PGPy==0.6.0` 拿 sdist + 其依赖 wheel；补 `cffi`(cp311)、`cryptography`(abi3 跨版本)、`setuptools/wheel`(构建后端)。
- **已用两个干净 venv（3.13 + 3.11.6）真实 `--no-index` 离线安装验证通过**：PGPy 从 sdist 成功构建为 wheel 并安装，全部 11 个依赖就位。
- 离线安装必须经「两步法」：① `pip install --no-index --find-links offline_wheels setuptools wheel` ② `pip install --no-index --no-build-isolation --find-links offline_wheels -r requirements.txt`（否则 PGPy 源码包构建会因隔离环境缺失 setuptools 失败）。
- 附 `offline_wheels/README.md`（含安装/启动/平台说明）与 `install_offline.bat`（一键脚本，需先激活 .venv）。
- 仅 Windows x64；若离线机为 Linux/3.8 需另行下载对应 wheel。

## 离线机 venv 报 ensurepip 失败 / 无 pip 的兜底方案
- 用户在离线机 `python -m venv .venv` 报 `ensurepip` non-zero exit 1 —— 根因：离线机 Python 是**嵌入式(embeddable zip)或安装时未勾选 pip**，`venv` 找不到 pip 引导包。
- **补 `pip-26.1.2-py3-none-any.whl` 到 offline_wheels**（py3-none-any，与版本无关；现 48 文件 / 22MB）。
- **验证通过的无 pip bootstrap 路径**（本机用 `--without-pip` venv + 解压 pip wheel 模拟）：
  `python -m venv --without-pip .venv` → 解压 `pip-*.whl` 到 `.venv/Lib/site-packages` → `python -m pip --version` 即可用 → 再两步法装依赖（PGPy 仍成功源码构建）。
- `install_offline.bat` 已升级为**自动判断**：检测 `python -m pip --version`，有则用（全局/已激活 venv），无则自动 `--without-pip` venv + 解压 pip wheel 引导。README 同步更新。
- 给用户的实操优先「跳过 venv 全局装」：若 `python -m pip --version` 有输出，直接全局 `pip install --no-index --find-links ..\offline_wheels ... -r requirements.txt`（离线机专职跑此项目，全局装无妨）；若报 `No module named pip` 才需重拷含 pip wheel 的 offline_wheels 走 venv 引导。
