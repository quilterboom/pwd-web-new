## 空态文案 bug 修复（2026-07-13 下午）
- 现象：密码页「暂无密码记录，点击右上角「新增密码」开始。」**有数据时也展示**。根因：该 banner 原用 `v-if="!state.entries.length"`，而 `state.entries` 是 store 里 `loadEntries()` 取的**全量快照**，与密码页实际驱动表格的**分页结果**（`fetchEntries()` 写入的 `entries`/`entriesTotal`）是两套数据源，易因刷新时序/筛选导致 `state.entries` 为空而表格有数据 → banner 误显；同时「无匹配结果」行也误用了 `state.entries.length`。
- 修复（`PasswordPanel.vue`）：空态完全改由**分页结果**驱动——新增 `loaded`(首次拉取完成)`、`hasFilter`(分组/系统/关键字任一非空) 两个 computed/ref；banner 改 `v-if="loaded && entriesTotal === 0 && !hasFilter"`；「无匹配结果」行改 `v-if="loaded && !entries.length && (entriesTotal > 0 || hasFilter)"`。效果：有数据（含筛选后无结果）绝不再显示新增引导 banner。
- 前端 `npm run build` OK（新产物 `index-Du-WbS5x.js`）；已 `build_image.sh` 重建镜像 → 新镜像 `c7450c14d623`（219MB，amd64）+ `password_manager_image.tar`（216MB）。
- 验证：容器启动后 `GET /` 引用 `index-Du-WbS5x.js`，且编译产物中 banner 守卫确为 `loaded && entriesTotal===0 && !hasFilter`（`b.value&&p.value===0&&!M.value`），空态文案不再出现在服务端 HTML（客户端按条件渲染）。临时容器 pm-emptycheck 已清理。
